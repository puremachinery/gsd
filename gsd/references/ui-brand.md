# UI Brand

Use this only when the project includes a user-facing interface.

## Principle

Make the interface feel intentional. Avoid generic layouts, default component-library aesthetics, and visual choices that do not support the product's character.

## What To Decide Early

- visual mood and tone
- typography direction
- color system
- spacing density
- motion style
- mobile and desktop behavior

## Guardrails

- prefer a clear visual direction over neutral default styling
- choose typography on purpose
- use color to communicate structure and emphasis, not decoration alone
- keep motion meaningful and sparse
- ensure the design still works on small screens
- preserve the existing design language if working inside an established product

## Avoid

- interchangeable dashboard styling
- flat, characterless defaults
- decorative motion without purpose
- accessibility regressions caused by contrast or tiny hit targets

## If No UI Exists

Ignore this guide and continue with the rest of the planning flow.
